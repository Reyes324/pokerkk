# 牌诀功能设计文档

**日期：** 2026-06-28  
**状态：** 已批准，待实现

## 背景与目标

猫店德州结算目前专注于结算核心功能。本次新增「牌诀」模块，为牌局中感到迷茫的玩家提供随机抽取德州扑克格言的功能，起到醍醐灌顶的作用。

---

## 功能范围

### 本次实现
1. **底部导航栏**（2 Tab）：结算 + 牌诀
2. **牌诀 Tab**：随机翻牌抽取格言（50 张）

### 不在本次范围
- 概率查询模块（未来新 Tab）
- 卡片分类 / 搜索
- B 方案「神谕召唤」动效（未来升级方向）

## 导航结构

### 底部导航栏
- 新增固定底部导航，位于现有 P&L 浮动结算条**下方**
- 共 2 个 Tab，未来可扩展至 3-4 个

| Tab | 图标 | 默认 |
|---|---|---|
| 结算 | 筹码图标 | ✅ 是 |
| 牌诀 | 扑克牌图标 | 否 |

- 记录入口保持现有右上角按钮，**不移入底部导航**
- 底部导航高度约 56px，背景使用 `--surface`，顶部 `1px solid var(--line)` 分割线
- 激活 Tab 用 `--brand` 色高亮图标和文字，未激活用 `--ink-3`

### 页面切换
- Tab 切换时对应内容区域显示/隐藏（不跳转页面，单页切换）
- 结算 Tab：现有主页内容（`#main-view`）
- 牌诀 Tab：新知识卡片视图（`#paijue-view`）

## 牌诀 Tab 交互设计

### 布局
```
┌─────────────────────────┐
│       app-header        │  固定顶部
├─────────────────────────┤
│                         │
│      [  卡片背面  ]      │  垂直居中
│                         │
│    点击翻牌看玄机 👆     │  提示文字
│                         │
│   ┌─────────────────┐   │
│   │   再抽一张       │   │  按钮
│   └─────────────────┘   │
│                         │
├─────────────────────────┤
│     P&L 浮动结算条      │  现有，不变
├─────────────────────────┤
│    [结算]    [牌诀]     │  底部导航
└─────────────────────────┘
```

### 状态机

**未翻牌状态（初始）**
- 屏幕中央显示卡片背面（深色卡面 + 简单装饰图案，如猫爪或扑克花色）
- 卡片下方显示提示文字：「点击翻牌」
- 「再抽一张」按钮：灰色/禁用（尚未翻开时无意义）

**已翻牌状态**
- 卡片翻转为正面，显示格言文字，居中大字
- 「再抽一张」按钮激活，点击后：
  1. 卡片翻回背面
  2. 随机选取新格言（不重复上一张）
  3. 自动再翻开（或等待用户点击）

### 翻牌动画
- CSS `transform: rotateY(180deg)`，持续 `0.45s`，`ease-in-out`
- 卡片背面：`rotateY(0deg)`（正常）
- 卡片正面：`rotateY(180deg)`，用 `backface-visibility: hidden` 防止穿透
- 不使用 JS 动画库，纯 CSS

### 卡片视觉规格
- 尺寸：宽 `min(300px, 80vw)`，高 `auto`（约 1.4:1 比例）
- 背面：`background: var(--surface)`，中央印花纹（SVG，扑克花色 ♠ 或猫爪）
- 正面：`background: var(--brand)`（深色品牌色），格言文字白色，`font-size: 20px`，`font-weight: 600`，`Noto Sans SC`，居中
- 圆角：`var(--r16)`
- 阴影：`var(--sh8)`

## 数据

### 卡片内容存储
- 新建 `js/cards.js`，导出一个字符串数组 `PAIJUE_CARDS`
- 50 条内容由用户提供，写入该文件
- 纯静态，不使用 Firebase

```js
// js/cards.js
const PAIJUE_CARDS = [
  '翻牌前少入池，翻牌后多弃牌。',
  '价值下注要薄，诈唬要厚。',
  // ... 共 50 条
];
```

### 随机抽取逻辑
- 每次「再抽一张」从 `PAIJUE_CARDS` 随机取一条
- 排除上一张（避免连续抽到相同格言）
- 不做已抽过的去重（50 张足够随机，不需要复杂状态）

- 仅结算 Tab 需要此间距；牌诀 Tab 内容全屏居中，不需要额外 padding
